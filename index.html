OpenAI API Changelog2026-07-22 00:00官方确认

OpenAI API 平台新增组织和项目硬性消费限额

OpenAI API 平台为组织和项目新增硬性消费限额（hard spend limits）功能，可设置月度消费上限，当追踪到的消费达到限额时，受影响的 API 请求将返回 429 错误。同时提供消费提醒（spend alerts）功能，在流量中断前进行通知。

业务影响

若未设置消费限额或限额过低，可能导致 API 调用在月中被中断（返回 429），影响依赖 OpenAI API 的业务连续性和用户体验；需评估现有消费模式并合理配置限额与告警。

需要行动

评估组织/项目的 API 消费情况，设置合理的月度硬性消费限额，并启用消费提醒以避免意外中断；更新相关监控与告警机制以应对 429 错误。
